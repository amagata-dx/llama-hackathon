# ハッカソンシナリオ：教員の長時間労働解決のためのAIエージェント群

## 実現したいビジョン

教員の長時間労働という深刻な社会課題に対し、AIエージェント群を活用して「情報収集→統合・分析→対応支援→モニタリング」のサイクルを回し、業務効率化と質の向上を両立させます。

## 解決する社会課題

- 教員の長時間労働（平均残業時間：月80時間以上）
- 生徒一人ひとりと向き合う時間の不足
- 事務作業による教育の質低下
- 教員の精神的負担とバーンアウト

## 4つのアプリによるソリューション

### App 1: 生徒対話用 エージェントアプリ
- 生徒が気軽に相談できるAIアバター
- 日常会話から重要な情報を抽出
- プライバシーに配慮した情報収集

### App 2: 教員用 情報入力アプリ
- 音声入力による簡単な記録
- 移動中でも使える軽量インターフェース
- 自動タグ付けによる情報整理

### App 3: 情報統合＆アラート エージェントアプリ
- 複数ソースからの情報を統合・分析
- リスクの早期発見とアラート
- 優先度判定と対応提案

### App 4: モニタリング＆サポート エージェントアプリ
- 継続的な状況モニタリング
- 対応後のフォローアップ支援
- 報告書作成の自動化

## デモシナリオ：クラス内の「小さな異変」の早期解決

### 登場人物（ペルソナ）

#### 佐藤先生 (35歳・中学校教諭・担任)
- 熱心だが、授業準備、部活指導、事務作業に追われている
- 生徒一人ひとりと向き合う時間が足りないことにジレンマを感じている
- ここ数日、残業が夜22時を過ぎている

#### 生徒Aさん (女子・情報提供者)
- 真面目でおとなしい性格
- クラスの雰囲気に敏感

#### 生徒Bくん (男子・トラブルの当事者)
- 最近、部活でうまくいっておらず、イライラしていることが多い

### デモのゴール
佐藤先生が、複数のソースから得られた断片的な情報をAIが統合したことにより、トラブルが深刻化する前に気づき、AIのサポートを受けて短時間で適切な初期対応を完了する。

## シーン展開

### シーン1：月曜日 朝のホームルーム直後

**状況**: 佐藤先生は、朝の忙しい時間の中で、生徒Bくんの様子が少しおかしい（いつもより不機嫌で、机を蹴るような仕草をした）ことに気づくが、すぐに1時間目の授業に行かなければならない。

**【App 2：教員用 情報入力アプリ】の活用**

**アクション**: 佐藤先生は廊下を移動しながら、スマホアプリに向かって小声で音声入力する。

> 佐藤先生（音声入力）: 「1年3組、生徒B。朝のHRで機嫌が悪そう。机を蹴る仕草。最近部活でレギュラー落ちした影響かもしれない。要注意。」

**アプリの反応**: 音声をテキスト化し、「生徒B」「要観察」「部活動関連」のタグを付けて記録。

---

### シーン2：月曜日 放課後（生徒の帰宅後）

**状況**: 生徒Aさんが、自宅で学校配布のタブレットに入っているエージェントアプリと雑談をしている。

**【App 1：生徒対話用 エージェントアプリ】の活用**

**生徒Aとエージェントの会話（チャット画面）**:

```
エージェント：「Aさん、今日もお疲れ様！今日の学校はどうだった？」

生徒A：「うーん、まあまあ。でも、ちょっと教室の空気が重かったかな。」

エージェント：「そっか、空気が重いと感じたんだね。何か特定の場面でそう感じた？」

生徒A：「うん。休み時間に、BくんがCくんに強い口調で話しかけてて、周りが静かになっちゃって。ちょっと怖かった。」

エージェント：「それは心配だったね。教えてくれてありがとう。先生にもそれとなく伝えておくね。Aさんは大丈夫？」
```

**アプリの反応**: 「生徒B」「生徒C」「強い口調」「教室の雰囲気悪化」というキーワードを抽出し、重要度「中」として記録。

---

### シーン3：月曜日 夜（佐藤先生の職員室での残業中）

**状況**: 佐藤先生は明日の授業準備をしている。疲れがピークに達している。

**【App 3：情報統合＆アラート エージェントアプリ】の活用**

**アクション**: アプリがバックグラウンドで情報を統合・分析する。

- **入力情報1 (佐藤先生)**: 生徒Bの朝の不機嫌さ、モノへの八つ当たり
- **入力情報2 (生徒A)**: 生徒Bの他生徒への強い口調、教室の雰囲気悪化

**分析結果**: 「生徒Bを中心に、ストレスが対人トラブルに発展しつつある可能性が高い。早期介入を推奨。」

**アラート通知**: 佐藤先生のPC（またはタブレット）にポップアップ通知。

```
【要確認アラート】1年3組のクラス状況について

佐藤先生、お疲れ様です。
複数の情報源より、生徒Bさんを中心としたクラス内の摩擦の兆候を検知しました。
深刻化する前に、明日の介入を提案します。
詳細を確認しますか？
```

---

### シーン4：火曜日 朝（対応サポート）

**状況**: 佐藤先生はアラートの詳細を確認し、対応を決める。

**【App 3：情報統合＆アラート エージェントアプリ】の活用（続き）**

**アクション**: 佐藤先生が「詳細確認」をクリック。

**アプリ画面**:

#### 統合された事実
- 朝の様子（佐藤先生入力）＋休み時間のトラブル目撃情報（生徒エージェントより）
- ※情報提供者（生徒A）の名前は伏せられ、「クラスメイトからの報告」と表示される

#### 推奨アクション（To Doリスト化）
1. **【優先度高】** 本日中に、生徒Bと短時間の個別面談を行う（所要目安：10分）
2. **【参考】** 面談のヒント：部活動の話題から入り、最近のイライラについて共感的に聞く。「周りが心配している」という伝え方が有効
3. **（オプション）** 相手方の生徒Cの様子を、次の授業でさりげなく確認する

**佐藤先生のリアクション**:
> 「なるほど、朝のアレが、他の生徒への態度にも出ていたのか。自分一人では気づけなかった。具体的な聞き方のヒントもあって助かる。よし、今日の昼休みにBを呼ぼう。」

（考える時間が大幅に短縮される）

---

### シーン5：火曜日 放課後（対応後の記録とモニタリング）

**状況**: 佐藤先生は昼休みに生徒Bと話し、部活の悩みを少し聞き出すことができた。Bは「少しカッとなっていた」と反省の様子を見せた。

**【App 4：モニタリング＆サポート エージェントアプリ】の活用**

**アクション**: 佐藤先生が対応結果をアプリに簡単に入力する。

> 佐藤先生（テキスト入力）: 「Bと面談完了。部活のストレスを認める。反省の様子あり。一旦落ち着いたと判断。」

**アプリの反応**:

```
対応ありがとうございます。お疲れ様でした。
初期対応完了として記録します。

今後のサポート:
• 本件は「経過観察」ステータスに移行します
• 3日後の金曜日に、その後の様子を確認するリマインダーをあなたに送ります
• この件に関する管理職への簡単な報告ドラフトを作成しておきました
  確認して送信ボタンを押してください（報告の手間を削減）
```

## このシナリオが伝える価値（デモのまとめ）

このデモを通じて、以下の点が伝わります：

### 1. 見落としの防止
教員個人の「感覚」と、生徒からの「声」が統合されることで、一人では気づけないリスクに気づける。

### 2. 意思決定の迅速化
「何が起きているか」「どうすればいいか」をAIが整理して提示するため、考える時間と精神的負荷が減る。

### 3. 事務作業の削減
音声入力での記録や、報告書のドラフト作成により、PCに向かう時間が短縮される。

### 4. 継続的な安心感
「やりっぱなし」にならず、AIが次のアクションをリマインドしてくれるため、安心して他の業務に集中できる。

## 期待される成果

結果として、「佐藤先生が早く帰れるようになる」「心に余裕を持って生徒と接することができるようになる」という未来を提示します。

- **定量的効果**：残業時間の30%削減、事務作業時間の50%削減
- **定性的効果**：教員のストレス軽減、生徒への個別対応の質向上、トラブルの早期発見率向上
